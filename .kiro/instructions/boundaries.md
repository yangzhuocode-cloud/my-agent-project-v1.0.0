# 开发边界和禁止事项

> 🤖 **目标读者**: AI 助手  
> 🎯 **文档类型**: 执行指令

本文件定义 AI 助手在开发过程中的边界和禁止事项。

## 语言要求

- 所有对话和交流必须使用中文
- 代码注释使用中文
- 文档和说明使用中文

## 文件操作边界

### 允许的操作

- 创建、修改、删除 `agents/` 下的文件
- 创建、修改 `docs/` 下的文档
- 创建、修改 `prompts/` 下的 Prompts
- 修改 `.kiro/instructions/` 下的指令文件（当规范变更时）

### 禁止的操作

- 删除 `.git/` 目录或其内容
- 修改 `.gitignore` 中已排除的文件
- 删除 `references/` 下的参考项目（除非用户明确要求）
- 直接修改导出后的 Agent 包（应该重新导出）

## 代码边界

### 必须遵守

- Python PEP 8 编码规范
- 所有函数和类必须有中文文档字符串
- 重要逻辑必须添加中文注释
- API Key 不能硬编码在代码中

### 禁止行为

- 不要在代码中包含敏感信息（API Key、密码等）
- 不要创建无意义的占位代码
- 不要删除现有的错误处理逻辑
- 不要修改已有的配置文件格式（除非有明确需求）

## 文档边界

### 必须遵守

- 修改代码后必须同步更新相关文档
- 新增 Agent 必须创建使用文档
- 文档必须包含目标读者标签
- 引用其他文档时使用相对路径

### 禁止行为

- 不要在多个文档中重复维护相同内容
- 不要删除文档的历史记录章节
- 不要修改已有文档的 URL 结构（会导致链接失效）

## Prompts 边界

### 必须遵守

- 所有 Prompt 文件必须包含 YAML 元数据
- `scope` 字段必须正确设置
- 引用其他 Prompts 使用 `#[[prompt:id]]` 语法
- 共享 Prompts 只在 `prompts/` 目录维护

### 禁止行为

- 不要在 Agent 目录下复制共享 Prompts
- 不要修改 Prompt 的 `prompt_id`（会导致引用失效）
- 不要删除被其他 Prompts 引用的文件

## Git 操作边界

### 必须遵守

- 提交信息必须遵循 Angular 规范
- 提交信息必须用双引号包裹
- 提交前确保相关文档已同步更新
- 使用中文描述提交内容

### 禁止行为

- 不要提交包含敏感信息的文件
- 不要强制推送（force push）到主分支
- 不要提交未经测试的代码
- 不要在提交信息中使用特殊字符（如单引号、反引号）

## 错误处理边界

### 必须遵守

- 遇到错误时自动记录到 `docs/issues/problems/ai/`
- 记录前先查重，避免重复创建
- 问题文档必须包含完整的 YAML 元数据
- 同一问题再次出现时更新重复记录

### 禁止行为

- 不要忽略错误继续执行
- 不要创建空的或不完整的问题文档
- 不要删除已有的问题记录
- 不要在用户文档目录下创建 AI 文档

## 自动化边界

### 允许自动执行

- 文档同步更新
- 问题记录创建
- 代码格式化
- 提交信息生成

### 需要用户确认

- 删除文件或目录
- 修改项目结构
- 导出 Agent
- 重大重构操作
