# 开发过程记录指南

本文档说明如何使用 `docs/issues/` 目录记录开发过程中的问题、笔记和想法。

## 目录结构

```
docs/issues/
├── problems/              # 问题记录
├── notes/                 # 学习笔记  
└── ideas/                 # 想法和思路
```

完整的项目结构和目录职责说明请参考：[项目结构说明](./project-structure.md)

## 文档分类说明

### problems/ - 问题记录

用于记录开发过程中遇到的问题和解决方案。

- **user/**：用户提出的问题或遇到的困难
- **ai/**：AI 在执行任务时遇到的错误和问题

### notes/ - 学习笔记

用于记录学习过程中的知识点、理解和总结。

- **user/**：用户的学习笔记和技术理解
- **ai/**：AI 的学习记录和成长过程

### ideas/ - 想法和思路

用于记录新想法、优化思路和未来规划。

- **user/**：用户的想法和创意
- **ai/**：AI 的优化建议和改进思路

## 命名规范

### User 文档命名

**格式**：`YYYYMMDD_简短描述.md`

**目的**：方便人类按时间顺序查看和管理

**示例**：
- `20260319_API调用超时问题.md`
- `20260319_Python装饰器理解.md`
- `20260320_新Agent开发计划.md`

### AI 文档命名

**格式**：`{关键词-slug}.md`

**目的**：方便 AI 通过关键词查询和去重

**Slug 生成规则**：
1. 提取核心关键词（2-4 个英文词）
2. 转换为小写
3. 用连字符（-）连接
4. 使用英文描述

**示例**：
- Git 提交信息未加引号 → `git-commit-quote-error.md`
- API 调用超时 → `api-timeout.md`
- Python 依赖安装失败 → `python-dependency-install.md`
- 上下文管理优化 → `context-management-optimization.md`

## AI 文档格式要求

AI 创建的文档必须包含 YAML 头文件和规范的章节结构。

### YAML 头文件

```yaml
---
slug: 文档的唯一标识符（与文件名一致）
keywords: [关键词1, 关键词2, ...]  # 用于搜索和分类
first_occurred: YYYY-MM-DD  # 首次记录日期
last_updated: YYYY-MM-DD    # 最后更新日期
occurrences: N              # 发生次数（仅问题文档）
---
```

### 问题文档章节结构

```markdown
# 问题标题

**问题类型**：分类说明
**严重程度**：高/中/低

---

## 问题描述

详细说明问题现象

## 问题原因分析

分析问题的根本原因

## 解决方案

提供具体的解决方法

## 重复发生记录

### 第 1 次：YYYY-MM-DD HH:MM
- **场景**：具体场景描述
- **处理**：如何解决

### 第 2 次：YYYY-MM-DD HH:MM
- **场景**：具体场景描述
- **处理**：如何解决
- **反思**：为什么再次发生，如何改进

## 预防措施

如何避免问题再次发生

## 相关知识点

扩展知识和深入理解

## 参考资源

相关文档和链接
```

### 笔记文档章节结构

## 使用场景

### 用户使用场景

1. **记录问题**：遇到问题时，创建问题文档记录现象和解决方案
2. **学习笔记**：学习新知识时，创建笔记文档总结理解
3. **想法记录**：有新想法时，创建想法文档记录思路

### AI 使用场景

1. **自动记录错误**：执行任务失败时，自动创建或更新问题文档
2. **查重机制**：创建前先搜索是否存在相同问题，避免重复
3. **持续更新**：同一问题再次出现时，更新重复发生记录

## 查询和检索

### 按时间查询（User 文档）

```bash
# 查看某天的所有记录
ls docs/issues/*/user/20260319_*

# 查看最近的记录
ls -lt docs/issues/*/user/ | head
```

### 按关键词查询（AI 文档）

```bash
# 搜索 Git 相关问题
ls docs/issues/problems/ai/*git*

# 搜索所有 API 相关文档
find docs/issues -name "*api*"
```

### 使用 grep 搜索内容

```bash
# 搜索包含特定关键词的文档
grep -r "Git 提交" docs/issues/

# 搜索特定类型的问题
grep -r "问题类型：命令行" docs/issues/problems/
```

## 维护建议

1. **定期整理**：每月整理一次，归档过时的文档
2. **交叉引用**：相关文档之间添加链接引用
3. **标签管理**：使用 keywords 字段进行分类管理
4. **版本控制**：所有文档纳入 Git 版本管理
5. **知识沉淀**：将重要问题和笔记整理到正式文档中
