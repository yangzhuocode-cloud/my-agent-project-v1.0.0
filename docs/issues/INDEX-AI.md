## 大模型与 Token 管理

### [大模型上下文长度中"k"的单位误解](./problems/ai/llm-context-length-k-unit.md)
- **关键词**: 大模型, 上下文长度, token, k单位, 256k, 1000, 1024
- **核心问题**: 错误地将 256k 理解为 256 × 1024 = 262,144 tokens
- **正确理解**: 256k = 256 × 1000 = 256,000 tokens（十进制千）
- **适用场景**: 配置模型上下文长度、计算 token 使用率、实现上下文管理

## Git 与版本控制

### [Git 提交信息未加引号错误](./problems/ai/git-commit-quote-error.md)
- **关键词**: git, commit, quote, shell, 命令行, 参数解析
- **核心问题**: 提交信息包含空格或特殊字符但未用引号包裹，导致 shell 参数解析错误
- **解决方案**: 始终使用双引号包裹提交信息
- **适用场景**: 执行 git commit 命令时

### [Git 提交信息特殊字符处理](./problems/ai/git-commit-special-characters.md)
- **关键词**: git, 提交信息, 特殊字符, 引号, 转义
- **核心问题**: 提交信息中的特殊字符需要正确转义
- **解决方案**: 使用双引号包裹，必要时转义特殊字符
- **适用场景**: 提交信息包含引号、反斜杠等特殊字符时

## 使用指南

### 何时查阅本索引

在以下情况下，应该查阅本索引：

1. **遇到错误时**: 检查是否是已知问题
2. **执行关键操作前**: 查看相关最佳实践
3. **配置系统参数时**: 确认单位和数值的正确性

### 如何使用

1. **按关键词搜索**: 根据问题相关的关键词找到对应文档
2. **阅读完整文档**: 点击链接查看详细的问题描述和解决方案
3. **应用解决方案**: 按照文档中的最佳实践执行操作

### 文档更新

- 每次遇到新问题并解决后，应创建新的问题文档
- 同一问题再次出现时，更新文档中的"重复发生记录"章节
- 定期检查文档的准确性和时效性

## 文档规范

所有 AI 问题文档应包含以下章节：

1. **YAML 头文件**: slug, keywords, first_occurred, last_updated, occurrences
2. **问题描述**: 清晰描述问题现象
3. **问题原因分析**: 深入分析根本原因
4. **解决方案**: 提供具体的解决方法和代码示例
5. **最佳实践**: 如何正确执行相关操作
6. **预防措施**: 如何避免问题再次发生
7. **重复发生记录**: 记录每次发生的场景和处理方式

详细规范请参考：[开发过程记录指南](./README.md)
