# 目录结构变更时难以定位需要更新的文档

**问题类型**：文档维护
**严重程度**：中
**状态**：待解决
**记录时间**：2026-03-19

---

## 问题描述

当项目的目录结构发生变更时（如新增文件、调整目录层级等），很难快速定位哪些文档中包含了目录结构的说明，需要同步更新。

### 具体场景

本次新增 `docs/git-workflow.md` 文件后，需要更新所有提到目录结构的文档：
- `.kiro/steering/main.md`
- `README.md`
- `docs/development-guide.md`
- `docs/faq.md`
- `docs/quick-start.md`

### 当前困难

1. **人工查找困难**：需要逐个打开文档检查是否包含目录结构说明
2. **AI 查找低效**：只能使用 `grepSearch` 全局搜索关键词，结果可能不精确
3. **容易遗漏**：没有系统化的方法确保所有相关文档都被更新
4. **维护成本高**：每次目录结构变更都需要花费大量时间定位和更新

## 问题影响

- 文档维护效率低
- 容易出现文档不一致的情况
- 增加了开发和维护的心智负担

## 当前解决方案

使用 `grepSearch` 搜索关键词：
```
目录结构|项目结构|文件结构|directory structure
docs/
```

然后逐个检查搜索结果，手动更新相关文档。

## 待探索的解决方案

### 方案 1：文档元数据标记

在文档开头添加元数据标记，明确说明文档中包含的内容类型：

```yaml
---
contains:
  - directory-structure
  - api-reference
  - configuration
---
```

优点：
- 可以快速过滤出包含目录结构的文档
- 元数据可以用于自动化工具

缺点：
- 需要手动维护元数据
- 增加了文档编写的复杂度

### 方案 2：目录结构集中管理

将所有目录结构说明集中到一个文档中，其他文档通过引用链接：

```markdown
详细的目录结构请参考 [项目结构说明](./project-structure.md)
```

优点：
- 单一数据源，只需更新一处
- 避免文档不一致

缺点：
- 用户需要跳转查看
- 可能影响阅读体验

### 方案 3：自动化脚本检测

编写脚本自动检测文档中的目录结构代码块：

```bash
# 搜索包含目录结构的 markdown 代码块
grep -r "^```$" docs/ | grep -A 10 "├──\|└──"
```

优点：
- 自动化程度高
- 可以集成到 CI/CD 流程

缺点：
- 需要维护脚本
- 可能有误报或漏报

### 方案 4：使用文档生成工具

使用工具自动从实际文件系统生成目录结构：

```bash
tree -L 3 -I 'node_modules|.git' > structure.txt
```

优点：
- 始终与实际结构一致
- 完全自动化

缺点：
- 需要集成到文档构建流程
- 可能需要额外的格式化处理

### 方案 5：建立文档更新清单

在 `.kiro/steering/main.md` 中维护一个明确的清单，列出哪些文档包含目录结构：

```markdown
## 包含目录结构的文档清单

当目录结构变更时，必须检查并更新以下文档：
- [ ] .kiro/steering/main.md
- [ ] README.md
- [ ] docs/development-guide.md
- [ ] docs/faq.md
- [ ] docs/issues-guide.md
```

优点：
- 简单直接
- 不需要额外工具

缺点：
- 清单本身也需要维护
- 依赖人工检查
