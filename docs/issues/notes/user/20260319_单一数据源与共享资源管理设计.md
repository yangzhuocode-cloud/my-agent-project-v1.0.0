# 单一数据源与共享资源管理设计

**日期**：2026-03-19  
**主题**：如何在项目中维护单一数据源，同时支持多个 Agent 共享使用

---

## 问题背景

在开发 AI Agents 项目时，遇到一个核心问题：

**场景**：项目中有一些规范文档（如 Git Commit 规范），需要被多个 Agent 使用。

**矛盾**：
- 开发时：希望只维护一份文件（单一数据源），避免重复维护
- 导出时：希望每个 Agent 都是独立的，包含所有必需的文件

**具体问题**：
- 如果只在项目根目录维护一份，导出 Agent 时就缺少这个文件
- 如果在每个 Agent 中都复制一份，修改时需要同步多处，容易不一致

---

## 讨论过程

### 方案探索

我们讨论了多种解决方案：

#### 方案 1：符号链接（Symlink）
- **思路**：在 Agent 目录中创建指向共享文件的符号链接
- **优点**：真正的单一数据源
- **缺点**：Windows 支持不好，导出时可能失效

#### 方案 2：构建时复制 + 导出脚本
- **思路**：开发时维护一份，导出时自动复制
- **优点**：跨平台兼容，导出后完全独立
- **缺点**：需要维护导出脚本

#### 方案 3：引用机制 + 运行时加载
- **思路**：Agent 中只保存引用路径，运行时动态加载
- **优点**：清晰的依赖关系
- **缺点**：实现复杂度高

#### 方案 4：模板 + 生成器（最终选择）
- **思路**：配置文件声明依赖 + 导出时自动处理
- **优点**：开发体验好，导出可靠，可扩展
- **缺点**：需要编写导出脚本（一次性工作）

### 关键决策

**选择方案 4 的原因**：
1. 开发时单一数据源，易于维护
2. 导出后完全独立，不依赖外部文件
3. 通过配置文件管理依赖，清晰明了
4. 支持未来扩展（版本管理、条件导出等）

## 最终设计

### 1. 目录结构设计

```
项目根目录/
├── prompts/                          # 项目级 Prompts（单一数据源）
│   ├── git_commit_angular_001.md    # 共享 Prompt (scope: shared)
│   └── prompt_generator_001.md      # 项目私有 Prompt (scope: private)
│
└── agents/my first agent/
    ├── config.json                   # 运行时配置
    ├── .export.json                  # 导出配置（不导出）
    ├── main.py
    └── prompts/                      # Agent 的 Prompts 目录
        └── agent.md                  # Agent 主提示词
```

### 2. 配置文件分离

**职责清晰**：
- `config.json` - 运行时配置（model、baseUrl 等）→ 需要导出
- `.export.json` - 导出配置（排除文件模式）→ 不导出

### 3. 元数据驱动

**核心思想**：通过文件头部的元数据控制共享行为

```yaml
---
prompt_id: git_commit_angular_001
scope: shared              # 共享范围
agents: ["*"]              # 所有 Agent 可用
version: 1.0.0
---
```

**Scope 类型**：
- `shared` - 全局共享，所有 Agent 可用
- `private` - 项目私有，不导出
- `selective` - 选择性共享，只给特定 Agent
- `agent_private` - Agent 私有，只属于该 Agent

### 4. 虚拟路径引用

**问题**：开发时和导出后的路径不一致

**解决方案**：使用虚拟路径标识

```markdown
# agents/my first agent/prompts/agent.md

你是 Git Commit 助手

## 遵循的规范

#[[prompt:git_commit_angular_001]]
```

**工作原理**：
- **开发时**：AI 看到引用，在 Agent 的 prompts/ 中找不到，向上查找到项目根目录
- **导出后**：导出脚本已将文件复制到 Agent 的 prompts/，AI 直接找到
- **关键**：引用语法保持不变，路径问题自动解决

### 导出脚本工作流程

```python
# 1. 复制 Agent 文件（排除 .export.json）
copy_agent_files(agent_path, export_path, exclude_patterns)

# 2. 扫描 prompts/ 目录，提取所有 #[[prompt:xxx]] 引用
prompt_refs = extract_prompt_references(agent_path / "prompts")

# 3. 从项目根目录读取被引用的 Prompt 文件
for prompt_id in prompt_refs:
    source = f"prompts/{prompt_id}.md"
    metadata = parse_prompt_metadata(source)
    
    # 4. 只复制 scope 为 shared 或 selective 的文件
    if metadata['scope'] in ['shared', 'selective']:
        copy_file(source, export_path / "prompts" / f"{prompt_id}.md")
```

## 关键技术点

### 1. 元数据规范
- 使用 YAML front matter 存储元数据
- 定义清晰的字段和验证规则
- 参考：[Prompt 元数据规范](../../prompt-metadata-spec.md)

### 2. 虚拟路径引用
- 使用 `#[[prompt:id]]` 语法
- 通过 prompt_id 引用，不依赖文件路径
- AI 自动解析，开发和导出后都能正确找到文件

### 3. 导出脚本
- 自动解析引用
- 读取元数据判断是否导出
- 保持引用语法不变

### 4. 配置分离
- 运行时配置 vs 构建时配置
- 清晰的职责划分

## 后续优化方向
