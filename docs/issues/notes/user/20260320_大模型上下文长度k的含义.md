# 大模型上下文长度中"k"的含义

**创建时间**: 2026-03-20  
**类型**: 技术笔记  
**标签**: #大模型 #上下文长度 #token #单位换算

## 问题背景

在开发 Agent 时，遇到了对大模型上下文长度单位"k"的理解误区，最初以为 256k = 256 × 1024 = 262,144 tokens（二进制千），但实际上应该是 256k = 256,000 tokens（十进制千）。

## 明确结论

**在大模型上下文长度中，256k 的 k 统一指 1,000（十进制千），不是 1024（二进制千）。**

### 正确理解

- **256k tokens = 256,000 tokens**（十进制）
- **不是** 256 × 1024 = 262,144 tokens

## 为什么不是 1024？

### 1. 上下文长度是计数单位，不是存储单位

- **Token 个数**：是计数概念，用于表示模型能处理多少个 token
- **字节（Byte）**：是存储概念，用于表示数据占用多少空间
- 只有在表示存储空间时，才会用到 1 KiB = 1024 bytes 的二进制单位

### 2. 行业惯例统一

主流大模型厂商在文档与宣传中均以 **1k = 1,000 tokens** 标注：

| 厂商 | 模型示例 | 上下文长度表示 |
|------|---------|--------------|
| OpenAI | GPT-4 | 8k = 8,000 tokens |
| OpenAI | GPT-4 Turbo | 128k = 128,000 tokens |
| Anthropic | Claude 3 | 200k = 200,000 tokens |
| 字节跳动 | 豆包 | 256k = 256,000 tokens |
| 阿里 | 通义千问 | 32k = 32,000 tokens |

### 3. 少数例外

- 仅少数开源框架或旧文档偶尔混用 1024
- 但主流商用与公开口径已统一为 1000

## 实际应用

### 代码中的正确写法

```python
# ✅ 正确：使用十进制千
MODEL_MAX_TOKENS = 256000  # 256k = 256,000 tokens

# ❌ 错误：使用二进制千
MODEL_MAX_TOKENS = 256 * 1024  # 262,144 tokens（不符合行业惯例）
```

### 注释说明

为了避免歧义，建议在代码中明确标注：

```python
MODEL_MAX_TOKENS = 256000  # 256k 上下文窗口（256k = 256,000 tokens）
```

## 相关概念对比

| 概念 | 单位 | k 的含义 | 示例 |
|------|------|---------|------|
| 上下文长度 | tokens | 1k = 1,000 | 256k = 256,000 tokens |
| 文件大小 | bytes | 1 KB = 1,000 bytes | 256 KB = 256,000 bytes |
| 内存大小 | bytes | 1 KiB = 1,024 bytes | 256 KiB = 262,144 bytes |

**注意**：
- KB（千字节）= 1,000 bytes（十进制，国际单位制）
- KiB（千字节）= 1,024 bytes（二进制，IEC 标准）

## 总结

1. **大模型上下文长度中，k = 1,000**（十进制千）
2. **这是行业统一惯例**，所有主流厂商都遵循
3. **不要与存储单位混淆**（存储中 KiB = 1,024）
4. **代码中应使用 256000 而不是 256 × 1024**
