# Agent 导出设计文档

**日期**：2026-03-19  
**主题**：Agent 导出功能的完整设计方案

---

## 设计目标

设计一个 Agent 导出功能，满足以下核心需求：

1. **开发时可用**：在项目中开发 Agent 时，能够正常运行和测试
2. **导出后可用**：导出的 Agent 是完全独立的，不需要修改任何代码即可运行
3. **路径一致性**：引用路径在开发和导出后保持一致
4. **自动化**：导出过程自动化，无需手动复制文件

---

## 核心挑战

### 挑战 1：路径不一致问题

**问题描述**：
```
开发时：
agents/my first agent/prompts/agent.md
  └─ 需要引用: ../../prompts/git_commit_angular_001.md

导出后：
my-first-agent/prompts/agent.md
  └─ 需要引用: ./prompts/git_commit_angular_001.md
```

路径完全不同！

**解决方案**：虚拟路径引用

使用 `#[[prompt:prompt_id]]` 代替相对路径：
- 开发时：AI 向上查找到项目根目录
- 导出后：导出脚本已复制文件到本地，AI 直接找到
- 引用语法保持不变

### 挑战 2：选择性导出

**问题描述**：
- 有些 Prompt 是共享的（应该导出）
- 有些 Prompt 是项目私有的（不应该导出）
- 如何自动判断？

**解决方案**：元数据驱动

通过 `scope` 字段控制：
```yaml
scope: shared       # 导出 ✅
scope: private      # 不导出 ❌
scope: selective    # 根据 agents 字段决定
scope: agent_private # 随 Agent 导出 ✅
```

### 挑战 3：依赖自动发现

**问题描述**：
- Agent 可能引用多个 Prompt
- 如何自动发现所有依赖？

**解决方案**：引用解析

扫描 Agent 的 prompts/ 目录，提取所有 `#[[prompt:xxx]]` 引用

## 设计方案

### 1. 目录结构设计

#### 开发时结构
```
项目根目录/
├── prompts/                          # 项目级 Prompts
│   ├── git_commit_angular_001.md    # scope: shared
│   └── prompt_generator_001.md      # scope: private
│
└── agents/my first agent/
    ├── config.json                   # 运行时配置（导出）
    ├── .export.json                  # 导出配置（不导出）
    ├── main.py
    ├── prompts/
    │   └── agent.md                  # 包含 #[[prompt:git_commit_angular_001]]
    └── references/
```

#### 导出后结构
```
my-first-agent/
├── config.json                       # 运行时配置
├── main.py
├── prompts/
│   ├── agent.md                      # 引用保持不变
│   └── git_commit_angular_001.md     # 自动复制过来
└── references/
```

**关键点**：
- `.export.json` 不导出
- 被引用的共享 Prompt 自动复制到 prompts/
- 引用语法 `#[[prompt:xxx]]` 保持不变

### 2. 配置文件设计

#### config.json（运行时配置）
```json
{
  "name": "my first agent",
  "model": "qwen2.5:14b",
  "baseUrl": "http://localhost:11434",
  "temperature": 0.7
}
```

**用途**：Agent 运行时读取  
**导出**：✅ 需要导出

#### .export.json（导出配置）
```json
{
  "exclude_patterns": [
    ".export.json",
    "__pycache__",
    "*.pyc"
  ]
}
```

**用途**：告诉导出脚本排除哪些文件  
**导出**：❌ 不导出

### 3. 元数据设计

每个 Prompt 文件包含 YAML front matter：

```yaml
---
prompt_id: git_commit_angular_001
scope: shared
agents: ["*"]
version: 1.0.0
description: Git Commit 规范
created: 2026-03-19
---
```

**关键字段**：
- `prompt_id`：唯一标识符，用于引用
- `scope`：控制导出行为
- `agents`：指定适用的 Agent

### 4. 引用语法设计

**虚拟路径引用**：
```markdown
#[[prompt:prompt_id]]
```

**示例**：
```markdown
# agents/my first agent/prompts/agent.md

你是 Git Commit 助手

## 遵循的规范

#[[prompt:git_commit_angular_001]]

## 你的任务
...
```

**优势**：
- 不依赖文件系统路径
- 开发和导出后语法一致
- AI 能够自动理解和解析

## 导出脚本设计

### 工作流程

```
1. 读取 .export.json 配置
   ↓
2. 复制 Agent 文件（排除配置的文件）
   ↓
3. 扫描 prompts/ 目录，提取 #[[prompt:xxx]] 引用
   ↓
4. 从项目根目录读取被引用的 Prompt 文件
   ↓
5. 解析元数据，检查 scope 字段
   ↓
6. 只复制 scope 为 shared 或 selective 的文件
   ↓
7. 生成独立的 Agent 包
```

#### 3. 判断是否导出
```python
def should_export_prompt(metadata: dict, agent_name: str) -> bool:
    """判断 Prompt 是否应该导出到指定 Agent"""
    scope = metadata.get('scope')
    
    if scope == 'private':
        return False
    if scope == 'shared':
        return True
    if scope == 'selective':
        agents = metadata.get('agents', [])
        return agent_name in agents or '*' in agents
    
    return False
```

### 1. YAML 解析

使用正则表达式解析简单的 YAML front matter：

```python
# 匹配 --- ... ---
match = re.match(r'^---\s*\n(.*?)\n---\s*\n', content, re.DOTALL)

# 解析 key: value
for line in yaml_content.split('\n'):
    if ':' in line:
        key, value = line.split(':', 1)
        # 处理数组 ["item1", "item2"]
        # 处理字符串 "value"
```

**优点**：
- 不依赖外部库
- 足够处理我们的元数据格式

### 2. 引用提取

使用正则表达式提取引用：

```python
pattern = r'#\[\[prompt:([^\]]+)\]\]'
matches = re.findall(pattern, content)
# 结果: ['git_commit_angular_001', 'other_prompt']
```

### 3. 文件复制

使用 `shutil.copy2` 保留文件元数据：

```python
shutil.copy2(source_file, dest_file)
```

### 4. 路径处理

使用 `pathlib.Path` 处理跨平台路径：

```python
from pathlib import Path

agent_path = Path("agents/my first agent")
export_path = Path("exports/my-first-agent")
```

## AI 如何理解引用

### 开发时（项目内）

```
1. AI 读取 agents/my first agent/prompts/agent.md
2. 发现 #[[prompt:git_commit_angular_001]]
3. 在当前目录 agents/my first agent/prompts/ 查找
   → 未找到
4. 向上查找到项目根目录 prompts/
   → 找到 git_commit_angular_001.md ✅
5. 读取并理解内容
```

### 导出后（独立 Agent）

```
1. AI 读取 my-first-agent/prompts/agent.md
2. 发现 #[[prompt:git_commit_angular_001]]
3. 在当前目录 my-first-agent/prompts/ 查找
   → 找到 git_commit_angular_001.md ✅
4. 读取并理解内容
```

**关键**：
- 引用语法相同
- AI 的查找逻辑：先本地，再向上
- 导出脚本确保本地有文件

## 最佳实践

### 1. Prompt 命名
- 项目级：`{功能}_{版本}.md`
- Agent 主提示词：固定为 `agent.md`

### 2. Scope 设置
- 通用规范 → `scope: shared`
- 项目工具 → `scope: private`
- Agent 专属 → `scope: agent_private`

### 3. 引用管理
- 使用虚拟路径，不用相对路径
- 在主提示词中集中管理引用
- 避免循环引用

### 4. 版本管理
- 使用语义化版本号
- 重大变更时更新版本
- 记录更新日期
