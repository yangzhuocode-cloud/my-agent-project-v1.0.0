# ADR-0001: 虚拟路径引用机制

**状态**: 已采纳  
**日期**: 2026-03-19  
**决策者**: 项目团队

## 背景和问题

在 Agent 开发过程中，主 prompt 需要引用其他子 prompt 文件。这带来了一个关键问题：

**开发时 vs 导出后的路径不一致**：
- 开发时：agent 在项目内，引用路径为 `../../prompts/git_commit_angular_001.md`
- 导出后：agent 独立运行，引用路径应为 `./prompts/git_commit_angular_001.md`

如果使用相对路径，会导致：
1. 开发时和导出后的引用路径不同
2. 需要在导出时修改代码中的路径
3. 维护困难，容易出错

## 考虑的方案

### 方案 1: 相对路径 + 导出时重写

**描述**: 开发时使用相对路径，导出脚本自动修改代码中的路径引用

```markdown
# 开发时
#[[file:../../prompts/git_commit_angular_001.md]]

# 导出后（脚本自动替换）
#[[file:./prompts/git_commit_angular_001.md]]
```

**优点**:
- 开发时和导出后都是真实路径
- 不需要额外的抽象层

**缺点**:
- 导出脚本需要解析和修改代码
- 容易出错，维护困难
- 不适合复杂的引用场景

### 方案 2: 路径解析器

**描述**: 使用虚拟路径标识符，运行时动态解析

```python
# 使用虚拟路径
prompt_path = "@shared/git_commit_angular_001.md"

# 路径解析器
def resolve_prompt_path(virtual_path):
    if is_development():
        return f"../../prompts/{virtual_path[8:]}"
    else:
        return f"./prompts/{virtual_path[8:]}"
```

**优点**:
- 灵活，支持多种路径类型
- 可以在运行时动态切换

**缺点**:
- 需要维护路径解析器
- 增加了复杂度
- 需要判断是否在开发环境

### 方案 3: 虚拟路径 + 文件复制（推荐）

**描述**: 使用 `#[[prompt:id]]` 虚拟路径，通过 prompt_id 引用，导出时复制文件

```markdown
# main_prompt.md（开发和导出后内容完全一样）
你是 Git Commit 助手

#[[prompt:git_commit_angular_001]]

## 你的任务
...
```

**AI 查找逻辑**：
- 开发时：在 agent 的 prompts/ 下找 → 没找到 → 向上查找到项目根目录 → 找到
- 导出后：在 agent 的 prompts/ 下找 → 找到（导出脚本已复制）

**优点**:
- 引用语法统一，开发和导出后完全一样
- AI 自动查找，先找本地，找不到向上查找
- 维护简单，只需维护根目录的共享 prompt
- 不依赖文件系统路径，不会因目录结构变化而失效

**缺点**:
- 需要导出脚本支持虚拟路径解析
- AI 需要理解这种引用语法（但 LLM 天然支持）

## 决策

选择方案 3：虚拟路径 + 文件复制

**理由**:

1. **路径统一性**
   - 引用语法在开发和导出后完全一致
   - 不需要修改任何代码或配置

2. **AI 友好**
   - LLM 天然理解 `#[[prompt:id]]` 这种引用语法
   - 类似 Markdown 链接，语义清晰

3. **维护简单**
   - 通过 prompt_id 引用，不依赖相对路径
   - 目录结构变化不影响引用关系

4. **导出后独立**
   - 导出脚本自动复制引用的文件
   - 导出的 agent 完全独立，不依赖外部文件

## 影响

**正面影响**:

1. **开发体验提升**
   - 引用语法简洁明了
   - 不需要关心相对路径问题

2. **维护成本降低**
   - 引用关系清晰，易于追踪
   - 不会因路径变化而失效

3. **导出可靠性**
   - 自动处理文件复制
   - 导出的 agent 完全独立

**负面影响**:

1. **导出脚本复杂度**
   - 需要解析虚拟路径引用
   - 需要处理文件复制逻辑

2. **学习成本**
   - 需要理解虚拟路径的概念
   - 需要遵循 `#[[prompt:id]]` 语法规范

**需要的工作**:

1. 定义虚拟路径语法规范
2. 更新导出脚本支持虚拟路径解析
3. 创建示例文档说明使用方法
4. 更新现有 prompt 文件使用虚拟路径

## 实现细节

### 虚拟路径语法

```markdown
#[[prompt:prompt_id]]
```

- `prompt_id` 对应 prompt 文件的元数据中的 `prompt_id` 字段
- 不包含文件扩展名（自动添加 .md）

### 导出脚本逻辑

```python
import re

def extract_prompt_references(content):
    """提取所有 prompt 引用"""
    pattern = r'#\[\[prompt:([^\]]+)\]\]'
    return re.findall(pattern, content)

def export_agent(agent_name):
    # 1. 读取 main_prompt.md
    main_prompt = read_file(f"agents/{agent_name}/prompts/main_prompt.md")
    
    # 2. 提取所有引用
    prompt_refs = extract_prompt_references(main_prompt)
    
    # 3. 复制引用的文件到导出目录
    for prompt_id in prompt_refs:
        source = f"prompts/{prompt_id}.md"
        dest = f"exports/{agent_name}/prompts/{prompt_id}.md"
        copy_file(source, dest)
    
    # 4. 复制 main_prompt.md（内容不变）
    copy_file(
        f"agents/{agent_name}/prompts/main_prompt.md",
        f"exports/{agent_name}/prompts/main_prompt.md"
    )
```
