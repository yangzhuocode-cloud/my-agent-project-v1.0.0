# 设计理念

> 📖 **目标读者**: 人类开发者  
> 🎯 **文档类型**: 理解

本文档解释 AI Agents 项目的核心设计理念和架构决策。

## 项目愿景

构建一个灵活、可扩展的 AI Agent 开发框架，让开发者能够：
- 快速创建和部署 AI Agent
- 复用和共享 Prompts 和配置
- 独立导出 Agent 为完整的可运行包

## 核心设计原则

### 1. 单一数据源 (Single Source of Truth)

**理念**: 共享的内容只维护一份，通过引用机制使用。

**体现**:
- 项目级 Prompts 存放在 `prompts/` 目录
- Agent 通过 `#[[prompt:id]]` 引用，而非复制
- 导出时自动解析引用并复制

**为什么**: 避免多处维护相同内容，减少不一致性。

### 2. 开发与导出分离

**理念**: 开发时保持灵活性，导出时保证独立性。

**体现**:
- 开发时：Agent 引用项目级资源
- 导出后：Agent 包含所有必需资源，完全独立

**为什么**: 开发时便于共享和更新，导出后便于分发和部署。

### 3. 人机分离的文档体系

**理念**: 人类和 AI 需要不同类型的文档。

**体现**:
- `docs/` - 给人类看的理解性文档
- `.kiro/steering/` - 给 AI 看的执行性指令

**为什么**: 
- 人类需要理解"为什么"和背景
- AI 需要明确的"做什么"和边界

### 4. 渐进式开发记录

**理念**: 记录开发过程，沉淀知识。

**体现**:
- `docs/issues/` 记录问题、笔记、想法
- User/AI 分离，命名规范不同
- 支持查重和持续更新

**为什么**: 
- 避免重复犯错
- 积累开发经验
- 便于回溯和学习

## 设计原则的应用

这些设计原则在项目的各个方面都有体现：

- **单一数据源** → Prompts 管理机制
- **开发与导出分离** → Agent 导出系统
- **人机分离** → 文档结构设计
- **渐进式记录** → Issues 管理机制
