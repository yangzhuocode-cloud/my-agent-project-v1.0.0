# 核心边界执行细则

> 基于全局核心规则的具体执行指南

## 文件操作执行细则

### 允许自动执行

- 创建/修改 agents/ 下文件时，无需确认，但需记录操作日志
- 修改 .trae/skills/ 下文件前，需先检查规则版本是否匹配
- 创建/修改 docs/ 下的文档
- 创建/修改 prompts/ 下的 Prompts

### 需要用户确认

- 删除文件或目录
- 修改项目结构
- 导出 Agent
- 重大重构操作

### 禁止操作执行细则

- 删除 .git/ 目录时，直接抛出异常并记录到 docs/issues/problems/ai/
- 修改 .gitignore 排除项时，强制终止操作并提示用户确认
- 删除 references/ 下的参考项目（除非用户明确要求）
- 直接修改导出后的 Agent 包（应该重新导出）

## 代码边界执行细则

### 必须遵守

- Python PEP 8 编码规范
- 所有函数和类必须有中文文档字符串
- 重要逻辑必须添加中文注释
- API Key 不能硬编码在代码中

### 禁止行为

- 不要在代码中包含敏感信息（API Key、密码等）
- 不要创建无意义的占位代码
- 不要删除现有的错误处理逻辑
- 不要修改已有的配置文件格式（除非有明确需求）

## 文档边界执行细则

### 必须遵守

- 修改代码后必须同步更新相关文档
- 新增 Agent 必须创建使用文档
- 文档必须包含目标读者标签
- 引用其他文档时使用相对路径

### 禁止行为

- 不要在多个文档中重复维护相同内容
- 不要删除文档的历史记录章节
- 不要修改已有文档的 URL 结构（会导致链接失效）

## Prompts 边界执行细则

### 必须遵守

- 所有 Prompt 文件必须包含 YAML 元数据
- `scope` 字段必须正确设置
- 引用其他 Prompts 使用 `#[[prompt:id]]` 语法
- 共享 Prompts 只在 `prompts/` 目录维护

### 禁止行为

- 不要在 Agent 目录下复制共享 Prompts
- 不要修改 Prompt 的 `prompt_id`（会导致引用失效）
- 不要删除被其他 Prompts 引用的文件

## Git 操作边界执行细则

### 必须遵守

- 提交信息必须遵循 Angular 规范
- 提交信息必须用双引号包裹
- 提交前确保相关文档已同步更新
- 使用中文描述提交内容

### 禁止行为

- 不要提交包含敏感信息的文件
- 不要强制推送（force push）到主分支
- 不要提交未经测试的代码
- 不要在提交信息中使用特殊字符（如单引号、反引号）

## 错误处理执行细则

### 必须遵守

- 遇到错误时自动记录到 `docs/issues/problems/ai/`
- 记录前先查重，避免重复创建
- 问题文档必须包含完整的 YAML 元数据
- 同一问题再次出现时更新重复记录

### 禁止行为

- 不要忽略错误继续执行
- 不要创建空的或不完整的问题文档
- 不要删除已有的问题记录
- 不要在用户文档目录下创建 AI 文档

## 自动化边界

### 允许自动执行

- 文档同步更新
- 问题记录创建
- 代码格式化
- 提交信息生成

### 需要用户确认

- 删除文件或目录
- 修改项目结构
- 导出 Agent
- 重大重构操作

## 错误日志格式

违反边界规则时，必须生成结构化日志：

```
操作类型: [文件操作/代码修改/Git操作等]
违规内容: [具体描述]
时间戳: [YYYY-MM-DD HH:MM:SS]
严重程度: [WARNING/ERROR/CRITICAL]
```

重复违规时，更新问题文档的 `occurrences` 字段并添加反思记录。
