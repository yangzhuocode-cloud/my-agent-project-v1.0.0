## 什么是 ADR？

ADR（Architecture Decision Record）是一种记录架构决策的轻量级文档格式。每个 ADR 记录一个重要的架构决策，包括：

- **背景**：为什么需要做这个决策
- **决策**：我们决定做什么
- **后果**：这个决策带来的影响（正面和负面）

## ADR 编号规则

- 使用三位数字编号：001, 002, 003...
- 按时间顺序递增
- 一旦创建，不修改编号

## 何时创建 ADR？

当遇到以下情况时，应该创建 ADR：

1. 选择技术栈或框架
2. 设计核心架构或数据结构
3. 改变重要的设计模式
4. 解决重大技术难题
5. 做出影响未来开发的决策

## ADR 模板

```markdown
# ADR-XXX: 决策标题

## 状态
提议 | 已接受 | 已废弃 | 已替代

## 背景
描述需要做决策的背景和问题

## 决策
描述我们的决策内容

## 后果
描述决策带来的影响（正面和负面）

## 替代方案
描述考虑过但未采用的其他方案

## 参考资料
相关链接和文档
```
